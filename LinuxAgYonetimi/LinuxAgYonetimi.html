Senaryo 1

Ubuntu Desktop için Ağ bağdaştırıcı ayarlarını NAT olarak ayarlayıp HOST (Kullandığınız) bilgisayardan SSH bağlantısı yapabildiğinizi (PORT yönlendirme ayarlarını yaptıktan sonra), ping yapamadığınızı gösterin.

Nat ayarı.

Port ayarı.

SSH kontrolü.

$ service --status-all | grep ssh

Ubuntu Desktop için SSH kurulumu.

SSH (Secure Shell), ağ üzerinde bulunan bir sunucuya bağlanmaya ve bağlanılan sunucu üzerinde komut çalıştırma, dosya transferi gibi işlemleri gerçekleştirmeye olanak sağlayan bir uzak sunucu bağlantı protokolüdür.

 
$ sudo apt install ssh

SSH kurulduktan sonra kontrol.

$ service --status-all | grep ssh

SSH aktifleştirme.

$ ssh localhost

HOST bilgisayardan SSH bağlantısı.

$ ssh localhost -p 14601 -l isik

Sanal makinanın Ip’sini öğrendim.

$ ip -c a

HOST bilgisayardan sanal makinaya Ping gönderememe.

$ ping 10.0.2.15

HOST bilgisayardaki dosya ismi soyadiniz.txt şeklinde olan bir dosyayı misafir (guest) işletim sistemine Misafir eklentileri yükledikten sonra kopyala yapıştır yaparak gerçekleştiriniz.

Misafir eklentisi kurulur.

HOST bilgisayardaki dosya ismi isik.txt olan txt dosyası, Ubuntu sanal makinaya yapıştırılır.

Dcore işletim sisteminin ağ bağdaştırıcı ayarlarını Köprü bağdaştırıcı olarak seçtikten sonra DC1 IP adresini bulun. SSH bağlantısı yapabilmeniz için gerekli yüklemeleri yaptıktan sonra HOST (Kullandığınız) bilgisayardan SSH bağlantısı ve ping yapıp/yapamadığınızı gösterin.

Dcore işletim sisteminin ağ bağdaştırıcı ayarlarını Köprü bağdaştırıcı olarak seçtim.

Dcore SSH kontrolü.

$ services

Dcore SSH kurulur.

Dcore ip adresi öğrenilir ve şifre atanır.

$ ifconfig
$ sudo passwd rezzan

Host bilgisayardan SSH bağlantısı kurulur.

$ ssh 192.168.1.23 -p 22 -l rezzan

Host bilgisayardan ping atanır.

$ ping 192.168.1.23

Ubuntu Server için yanlızca ana makine bağdaştırıcısı olarak ayarlama yaptıktan sonra IP adresini bulun. SSH bağlantısı yapabilmeniz için gerekli yüklemeleri yaptıktan sonra HOST (Kullandığınız) bilgisayardan SSH bağlantısı ve ping yapamadığınızı gösterin.

Ubuntu Server için yalnızca ana makine bağdaştırıcısı olarak ayarlama yapılır.

IP adresi bulunur.

$ ip -c a

Ubuntu Server OpenSSH kurulumu.

$ sudo apt-get install openssh-client
$ sudo systemctl start ssh
Bilgilerini görmek için: $ systemctl status ssh

HOST bilgisayardan SSH bağlantısı.

$ ssh 192.168.56.101 -p 22 -l rezzan

HOST bilgisayardan ping testi.

$ ping 192.168.56.101

Senaryo 2

userver1, userver2 ve userver3 için ağ bağdaştırıcı ayarlarını dahili ağ olarak ayarlayın. Bilgisayarların IP adreslerini statik olarak ayarlayın.

Üzerinde çalışacağımız bilgisayarların, daha net anlaşılması için hostnamelerini değiştiriyoruz. Bu komutları kullanarak açılan ekrandaki hostname’i silip yenisini yazıyor ve yeniden başlatıyoruz.

sudo nano /etc/hostname
sudo nano /etc/hosts
sudo reboot

Netplan yüklenir.

Netplan, tüm Ubuntu sistemlerinde ağ yapılandırmalarını basit bir şekilde yönetme ve yapılandırma misyonuna sahip olan Ubuntu 17.10’dan beri uygulanan bir komut satırı yardımcı programıdır.

 
$ sudo apt install netplan
$ sudo apt install netplan.io

Serverların ağ bağdaştırıcıları dahili ağ olarak ayarlanır.

Statik Ip ataması yapmak için bu komut ile dosya açılır.

$ sudo nano /etc/netplan/50-cloud-init.yaml

Açılan ekrana aşağıdaki gibi veri girilir.

network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
     dhcp4: no
     addresses: [192.168.4.3/24]
     gateway4: 192.168.4.1
     nameservers:
       addresses: [8.8.8.8,8.8.4.4]

Static ip ataması uygulaması ve kontrolü yapılır.

$ sudo netplan apply
$ sudo netplan --debug apply
$ ip -c a

Bu üç bilgisayardan birbirilerine ssh yapabildiğinizi ama HOST bilgisayardan bunlara SSH yapamadığınızı gösterin. ping komutunu kullanarak bu bilgisayarlar arasında haberleşmelerin olduğunu gösterin.

Serverlar arası ping kontrolü. userver1’den userver2 ve userver3’e ping yollanır.

$ ping 192.168.4.5

Serverlar arası SSH bağlantısı. userver1 ile userver2 ve userver3 arasında ssh bağlantısı kurulur.

$ ssh 192.168.4.5 -p 22 -l isik

HOST bilgisayardan, serverlara SSH denemesi.

$ ssh 192.168.4.3 -p 22 -l isik
$ ssh 192.168.4.4 -p 22 -l isik
$ ssh 192.168.4.5 -p 22 -l isik

userver1 bilgisayardaki dosya ismi adiniz.txt, soyadiniz.txt ve numaraniz.txt şeklinde olan bir dosyaları userver2 bilgisayarına gönderiniz. Gönderim işlemi için sFTP veya SCP protokolünü kullanmanız gerekir.

userver1 de gönderilmek için dosyalar oluşturulur.

$ touch rezzan.txt
$ touch isik.txt
$ touch 170202043.txt

userver1’deki dosyalar SCP bağlantısı ile userver2’ye gönderilir.

$ scp rezzan.txt isik@192.168.4.4:/home/isik
$ scp isik.txt isik@192.168.4.4:/home/isik
$ scp 170202043.txt isik@192.168.4.4:/home/isik

userver2 de, gönderilen dosyaların gidip gitmediği kontrol edilir.

$ ls

uPC1 , uPC2 , userver1 ve userver2 için router ayarları yapmak.

Kurulum komutu çalıştırılır.

$ sudo apt install quagga

Belgeler yüklenir.

$ sudo apt install quagga-doc

Bu komut ile açılan dosyanın ayarları yapılır.

$ sudo nano /etc/quagga/daemons

zebra=yes
bgpd=no
ospfd=yes
ospf6d=no
ripd=no
ripngd=no
isisd=no
babeld=no

Yapılandırma komutları yazılır.

$ cp /usr/share/doc/quagga-core/examples/vtysh.conf.sample /etc/quagga/vtysh.conf
$ cp /usr/share/doc/quagga-core/examples/zebra.conf.sample /etc/quagga/zebra.conf
$ cp /usr/share/doc/quagga-core/examples/bgpd.conf.sample /etc/quagga/bgpd.conf
$ sudo chown quagga:quagga /etc/quagga/*.conf
$ sudo chown quagga:quaggavty /etc/quagga/vtysh.conf
$ sudo chmod 640 /etc/quagga/*.conf

Hizmetleri başlatırız.

sudo service zebra start
sudo service bgpd start

Açılan dosyalar da düzenleme yapılır.

$ sudo nano /etc/bash.bashrc

export VTYSH_PAGER=more
$ sudo nano /etc/environment

VTYSH_PAGER=more

Servisimiz kuruldu mu kontrol edilir. Zebra ve bpgd görülür.

$ ls /etc/init.d/

uPC1 üzerinden veri paketlerinin userver1, userver2 ve uPC2 bilgisayarlarına ayrı ayrı nasıl gittiğinin ağ analizini yapınız.

Traceroute paketi kurulur.

Traceroute, bir IP ağı üzerindeki bir paketin kaynaktan hedefe giden yolunu izlemek için kullanılan bir ağ tanılama aracıdır. Traceroute ayrıca paketin hedefe giderken yaptığı her atlama için geçen süreyi de kaydeder.

 
$ sudo apt install traceroute

uPC1 üzerinden veri paketlerinin userver1, userver2 ve uPC2 bilgisayarlarına ayrı ayrı nasıl gittiğini gösteriniz. Bunun için uygun bir komut kullanmanızı gerekiyor.

uPC1 üzerinden userver1’ye ip adreslerini kullanarak veri paketlerinin gidişini test ettim.

$ ping 192.168.1.254
$ sudo traceroute 192.168.1.254

uPC1 üzerinden userver2’ye ip adreslerini kullanarak veri paketlerinin gidişini test ettim.

$ ping 192.168.2.254
$ sudo traceroute 192.168.2.254

uPC1 üzerinden uPC2’ye ip adreslerini kullanarak veri paketlerinin gidişini test ettim.

$ ping 192.168.2.1
$ sudo traceroute 192.168.2.1
